# ADR 0001: Milestone checklist instead of a workflow state machine

Job Tracking uses an ordered milestone checklist with sub-milestones for change orders rather than a formal workflow state machine with guarded transitions. Every job gets the same 12-milestone template copied in on creation; milestones are checked off independently without enforcing order or requiring prior milestones to be complete. Sub-milestone groups handle change orders (redo → requote → reapprove → re-enter production) and future parallel phases. This was chosen over a state machine because it is simpler to build and maintain, gives a granular within-production view the user needs, and accommodates change-order loops and parallel flows without adding transition logic — the same flexibility that would require significant state machine complexity. A state machine can be introduced later if tracking transitions and enforcing guards becomes necessary.
